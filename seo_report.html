Executive Summary

Melilotus offers premium handcrafted juttis, but the website's current technical foundation is hindering its visibility on search engines. While the aesthetic appeal of the brand is strong, the site suffers from critical mobile performance bottlenecks, missing fundamental on-page SEO tags, and a weak backlink profile. By addressing these technical barriers and implementing a targeted SEO content strategy, we can significantly improve organic rankings, drive high-intent e-commerce traffic, and increase overall sales conversions.

On-Page SEO Analysis

  • Heading Structure Needs Fix
    The first heading on the page is not an <H1> tag. Proper heading hierarchy is crucial for search engines to understand page context.
    Ensure the page has exactly one main <H1> tag at the top of the content structure, clearly defining the topic of the page.
  • JSON-LD Structured Data Missing
    No JSON-LD structured data was found on the page. This prevents rich snippets from appearing in Google search results.
    Implement Product, Organization, and WebSite schemas to help search engines index your brand and juttis correctly.
  • Open Graph (Social) Tags Incomplete
    Open Graph tags govern how URLs are displayed when shared on social media. Current setup is missing critical properties:
    og:title
    Melilotus | Premium Handcrafted Juttis
    og:description
    Shop the finest handcrafted and embellished juttis at Melilotus.in.
    og:type
    Missing
    og:url
    Missing
    og:image
    Failed to fetch (https://melilotus.in/logo.png)
    Define og:type (e.g., website or product), include the canonical og:url, and ensure the og:image URL points to a valid, accessible, high-resolution image.

Technical SEO Analysis

  • Canonical URL Missing
    No canonical URL tag is present in the rendered HTML. This can lead to duplicate content issues if the site is accessible via HTTP/HTTPS or with/without 'www'.
    Add a self-referencing canonical tag to the <head> section:
    <link rel="canonical" href="https://melilotus.in/" />
  • Crawling & Indexing Incomplete
    While a robots.txt file exists (https://melilotus.in/robots.txt), explicit indexation directives are missing from the page.

    • Robots Meta Tag: Missing
    • X-Robots-Tag HTTP Header: Missing
    Add a robots meta tag (e.g., <meta name="robots" content="index, follow">) to explicitly allow search engines to index the page and follow links.
  • XML Sitemap Missing
    No sitemap was found. A sitemap is essential for search engines to efficiently discover and crawl all URLs on the website.
    Generate an XML sitemap, link it inside your robots.txt file, and submit it to Google Search Console.
  • Hreflang Tags Missing
    Hreflang tags are not present. If you plan to target multiple languages or regions, this is required.
    If your store serves multiple countries (e.g., India, US, UK), implement hreflang tags to serve the correct regional variations. If it is purely single-region, this can be ignored for now.
  • HTTP to HTTPS Redirect Needs Fix
    The website is currently accessible via both HTTP and HTTPS versions. Search engines may treat these as two separate websites, causing duplicate content issues and diluting your SEO ranking power.
    Implement a site-wide 301 redirect from the HTTP version to the secure HTTPS version so that all traffic and search engine bots are forced to the secure site.
  • Facebook Sharing / Robots.txt Info / Normal
    When sharing the link on Facebook (https://www.facebook.com/share/1DZGon5LHF/), the status may show as "Blocked by robots.txt (Status Code 0)" by external crawling tools.

    Context: The crawling tool is restricted from accessing this URL because Facebook's own robots.txt file disallows it. This is entirely normal for external social media platforms protecting their servers from bots; it does not negatively impact your own site's SEO.

Additional Audit Findings

  • Weak Backlink Profile High Priority
    The site has a very low level of backlink activity (only 27 total backlinks from 22 referring domains). Search engines use backlinks as a strong indicator of authority.
    Execute a comprehensive link-building strategy to acquire high-quality, relevant backlinks from authoritative domains.
  • Slow Page Load & Mobile Performance Needs Fix
    The website loads slowly (Server Response: 1.8s, All Content: 4.7s) and scores poorly (62/100) on Google's Mobile PageSpeed Insights, despite a great Desktop score (97/100). The site is also using an outdated HTTP protocol instead of HTTP/2+.
    Enable HTTP/2+ protocol, reduce unused JavaScript, and optimize server response times to improve mobile load speeds.
  • Keyword Consistency & Distribution Missing
    Main target keywords are not effectively distributed across important HTML tags (Title, Meta Description, Headings).
    Ensure target keywords are naturally integrated into the Title tag, Meta Description, and Header tags.
  • Generative Engine Optimization (GEO) Missing
    The site is missing an Identity Schema (Organization/Person) and an llms.txt file. This limits the ability of LLMs and AI Search Engines to confidently understand the brand's entity structure.
    Implement Identity Schema and create an llms.txt file to improve AI search engine readability.
  • Missing Social Links & Tracking Info
    No associated social media profiles (Facebook, Instagram, X, YouTube, LinkedIn) are linked on the page. Furthermore, Analytics tools and Facebook Pixel were not detected.
    Link official social profiles and install Analytics / Facebook Pixel for tracking user behavior and retargeting.
  • Missing Mail Security Records Needs Fix
    The domain is missing DMARC and SPF mail records.
    Add DMARC and SPF records to your DNS settings to improve email deliverability and prevent domain spoofing.
  • Technical, Structural & Security Flags Needs Fix
    The spreadsheet highlights several specific technical and structural flags that need addressing:
    • Security Headers: Missing X-Content-Type-Options, X-Frame-Options, Content-Security-Policy, and Secure Referrer-Policy headers.
    • URL Formatting: Underscores and parameters were found in URLs (URLs should ideally use hyphens and be clean).
    • Content & Meta: Instances of duplicate Page Titles, Titles over 60 characters, duplicate Meta Descriptions, Soft 404 pages, and images missing size attributes.
    • Links: Internal outlinks missing anchor text.
    Implement the missing security headers in your server configuration, sanitize URL structures, ensure unique meta tags for every page, and add size attributes/anchor text where missing.
  • Profile Configuration Needs Fix
    The label "Account Type: customer" is currently visible on user profiles.
    Remove the "Account Type: customer" designation from the profile to streamline the interface and improve the user experience.

  • Mobile Performance Bottlenecks Critical Priority
    Category: Performance (34/100)

    The mobile experience is heavily bottlenecked by slow loading times. Key metrics flagged:
    • Total Blocking Time: 910 ms (High main-thread blocking)
    • Largest Contentful Paint (LCP): 28.6 seconds
    • Time to Interactive: 29.8 seconds
    • Network Chaining: Critical requests are chained, delaying render.
    Optimize your JavaScript execution, defer non-critical CSS/JS, and fix the LCP breakdown by ensuring the largest image or text block loads much earlier in the network waterfall.
  • Accessibility Compliance Needs Fix
    Category: Accessibility (85/100)

    While mostly accessible, a few elements prevent full screen reader compatibility and usability:
    • Buttons and links are missing accessible/discernible names (e.g., icon-only buttons).
    • Certain background and foreground color combinations do not meet sufficient contrast ratios.
    Add aria-label attributes to icon buttons, provide descriptive text for all links, and adjust color palettes to meet WCAG AA contrast standards.
  • Invalid Robots.txt Needs Fix
    Category: SEO (92/100)

    Lighthouse flagged the robots.txt file as invalid or improperly formatted.
    Validate the syntax of your robots.txt file using a testing tool and ensure there are no malformed rules blocking critical assets.

Growth Strategy & Recommendations

Beyond technical fixes, here is our strategic marketing roadmap for growing the Melilotus brand online.
  • Keyword Gap & Content Strategy Growth Opportunity
    Currently, the site has very thin textual content (215 words total). To capture high-intent buyers, we need to create targeted content around what your customers are actively searching for.
    Action: Launch optimized category pages and blog content targeting high-value long-tail keywords such as "embroidered bridal juttis," "comfortable ethnic footwear," and "how to style juttis with western wear."
  • Conversion Rate Optimization (CRO) Growth Opportunity
    Getting traffic is only the first step; converting visitors into buyers is crucial for ROI. Technical slowdowns and lack of visible trust signals currently interrupt the shopping experience.
    Action: Optimize the mobile checkout flow, display prominent trust badges (secure checkout, easy returns), showcase customer reviews directly on product pages, and ensure high-resolution product images load instantly.
